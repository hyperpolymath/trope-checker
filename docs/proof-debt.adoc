== Summary

*Net soundness-relevant escape hatches in this repository: 0.*

`check-trusted-base.sh` flags two `lean-sorry-or-axiom` markers — the Lean
axiom-audit doc-comment headings `## Axiom audit (sorry-free …)`, in which the
substring `sorry` (of `sorry-free`) trips the scanner's `\bsorry\b` pass. That
pass, *unlike* the scanner's Idris pass, applies *no comment exemption*, so an
audit comment that asserts the *absence* of `sorry` is itself counted as a hit.
Every flagged line is such an audit comment — a claim that the construct is *not*
used — not a use of it. No proof in `verification/proofs/` uses `sorry`, a user
`axiom`, `believe_me`, `assert_total`, a top-level `partial`, `unsafePerformIO`,
or `unsafeCoerce`; the Lean and Agda developments are `sorry`-free /
zero-`postulate`, audited separately (`verification/proofs/INTEGRATION-REPORT.adoc`).
The matches are therefore *false positives* of a substring scanner that is not
comment-aware for Lean — the same class of false positive recorded for the local
`proof-scan-dangerous` recipe in the integration report. The six Idris2
`believe_me` convention comments are listed below too for completeness, though the
scanner's Idris pass already exempts them.

== Standing position

* No escape hatch is admitted into the trusted base. The verified Idris2 core, the
  Lean grade-algebra ground truth, and the Agda metatheory are all axiom-clean for
  their claimed lemmas.
* The convention comments are retained deliberately: they document the prohibition
  at the point of work. The *root* fix for the scanner mismatch is to make
  `check-trusted-base.sh` comment-aware upstream in `hyperpolymath/standards`
  (solutions-at-source); this ledger records the local ground truth in the
  meantime.
* Should a genuine escape hatch ever be introduced, it MUST be added here under
  class (a), (b), or (c) with its justification before the proof is accepted.
