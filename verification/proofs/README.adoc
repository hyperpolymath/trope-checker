== Verification Commands

[source,bash]
----
just proof-check-all       # Run all proof checkers (per-file scan + dangerous-pattern scan)
just proof-check-idris2    # Idris2 only
just proof-check-lean4     # Lean4 only
just proof-check-agda      # Agda only
just proof-check-coq       # Coq only

# The two pinned developments need a real build (lake+Mathlib / an All.agda entry),
# not a bare per-file check — use these:
just verify-all            # grade-boundary + grade-factorisation (Lean) + Theorem B (Agda): build + honesty gate
just build-lean            # both Lean harnesses (grade-boundary + grade-factorisation) + axiom audits
just build-agda            # Agda Theorem B (--safe --without-K)
# the probe also has its own subproject harness:
( cd lean4/grade-factorisation && just falsify )   # sorry-free, axiom-clean, falsifier witnesses live
----

== Banned Patterns

The following MUST NOT appear in any proof file:

- `believe_me` (Idris2)
- `assert_total` (Idris2)
- `postulate` (Idris2/Agda)
- `sorry` (Lean4)
- `Admitted` (Coq)
- `unsafeCoerce` (Haskell)

These are banned as *real constructs*, not as words: the authoritative guarantees are
the provers themselves (`agda --safe` rejects postulates/`trustMe`/`TERMINATING`; the
Lean axiom audit rejects `sorryAx`), which cannot be evaded by comment tricks. The
`just proof-scan-dangerous` lint is a *comment-aware* pre-filter (it strips comments
before matching, so honest prose like "sorry-free" / "no postulate" is not flagged) and
is advisory, not a CI gate. CI enforcement is `panic-attack assail`.

== Adding New Proofs

1. Choose the appropriate prover (see `../../docs/status/PROOF-NEEDS.adoc`)
2. Create the `.idr`/`.lean`/`.agda`/`.v`/`.tla` file in the right directory
3. Ensure `%default total` (Idris2) or equivalent
4. Run the verification command
5. Update `../../docs/status/PROOF-STATUS.adoc`
