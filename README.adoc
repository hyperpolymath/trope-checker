== What this is

The tropechecker consumes a *Trope IR* — a labelled DAG of property-instances
(nodes) and graded effect-applications (edges) with a declared use-model floor —
composes the loss-shape grades, and returns a verdict: `p-sufficient` or
`p-insufficient`, with a *witness edge* when insufficient. The normative
semantics is the calculus in `spec/calculus.adoc`; this repository holds it, the
IR schema, the verified core, and the conformance suite.

[IMPORTANT]
.The checker is language-independent (HC-1)
====
The checker MUST NOT parse, import, execute, or depend on any source language.
It is a pure function _of the IR_, and the IR (its JSON schema) is the trust
boundary. A front-end's grade annotations are *claims*; the checker validates
their *composition*, not their origin. `{{LANG}}` (Haec) is the first front-end
that lowers to the IR — dogfooding without coupling — but a static analyser for
an existing language could emit the same IR and be checked by the same checker.
====

== The three firewalls (HC-2 — never let code launder them)

. *Soundness, not completeness.* A conservative `p-insufficient` is acceptable;
  claiming _less_ loss than occurred is never sound. No completeness is claimed.
. *Relative to faithful lowering.* The checker bounds IR-level loss by IR-level
  grades. It does NOT prove that a front-end's claim "this construct _is_ a
  `collapse`" is true — that is the front-end's lowering-correctness obligation
  (calculus O2), discharged per front-end, not here.
. *Veridicality is declared and propagated, not checked.* Detecting that a value
  claimed faithful is in fact false needs a ground-truth oracle the checker does
  not have (calculus O5). The checker guarantees only that deception does not
  compose away and that deceptive grades fail every honest floor.

== Deploy variant: prevent (R-2026-06-22)

Deception is treated as a *validation-boundary fault*. The three deceptive grades
(`fate=Falsified`, `bond=Misbound`, `merge=Conflated`) and any untagged merge are
not writable in source (HC-3) and are *malformed* in a valid Trope IR: the schema
rejects them at validation, naming the offending coordinate. The alternative
`detect` variant (admit them, catch at the verdict) is documented in the calculus
(§6.1) but is not the shipped profile.

== Run the checks

[source,sh]
----
just check        # schema validation + Idris2 typecheck + conformance suite
just trope-conformance   # just the IR→verdict conformance corpus
idris2 --typecheck verification/proofs/idris2/trope.ipkg   # the verified core
src/idris2/build/exec/tropecheck tests/conformance/fixtures/duck-collapse.ir.json
just verify-all   # grade-algebra ground truth (Lean) + Theorem B (Agda), both green + honesty gate
just build-lean   # both Lean harnesses (grade-boundary + grade-factorisation) + axiom audits
just build-agda   # just the Agda Theorem B metatheory (--safe --without-K)
----

The conformance suite (`tests/conformance/`) is the formal sibling of the
workbench's human _particularity trainer_: any implementation MUST reproduce its
verdicts and witness edges to claim conformance. "Portable" means
_means-the-same-everywhere_, enforced by the suite, not merely _runs-everywhere_.

== Where it sits — the three-repo picture

[%hardbreaks]
*Haec* (a language, one of many) — writes & type-checks programs; emits Trope IR.
        │  emits
        ▼
*Trope IR* — a graded-effect DAG (the wire format / trust boundary).
        │  consumed by
        ▼
*trope-checker* (this repo) — composes grades, checks each use-model, returns a verdict.

Both the language and the checker *realise* the *Particularity* vocabulary
(`github.com/hyperpolymath/trope-particularity-workbench`). See
`docs/trope-estate-architecture.svg`.

== Status

Alpha. The schema, executable checker, and conformance suite are green; the
verified core typechecks with no axioms behind its claimed lemmas. The grade-algebra
*honest/deceptive cancellation boundary* is now proved in Lean (sorry-free,
axiom-clean) and its metatheory developed in Agda (Theorem B). Grade soundness (the
full `g ⊑ ρ` theorem) and the *operational model for `fix`/routing* (the central open
problem, CO-1, gated by O4) remain the named open problems in
`docs/status/PROOF-STATUS.adoc`; see `verification/proofs/INTEGRATION-REPORT.adoc` for
the full proved-vs-missing picture. A Rust fast-core is drafted in `src/rust/`
(dep-free, validated against the Idris2 reference via the conformance corpus).
