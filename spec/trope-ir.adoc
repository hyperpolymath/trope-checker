= The Trope IR — normative interchange spec
:revnumber: 0.2
:revdate: 2026-07-07
:toc: macro
:sectnums:

[abstract]
This document specifies the *Trope IR*: the language-neutral interchange the
tropechecker consumes. It is normative for the wire format and the grade algebra;
the underlying type discipline is the calculus (`spec/calculus.adoc`), which this
document references rather than restates. The canonical wire format is the
JSON-Schema-validated document `schemas/trope-ir.schema.json` (the *prevent*
profile). The key words MUST/MUST NOT/SHOULD/MAY are per RFC 2119/8174.

toc::[]

== The document

A Trope IR document is a JSON object with `version` (`"0.2"`; `"0.1"` remains
accepted — see <<changes-from-0-1>>), `profile` (`"prevent"`), `nodes`, `edges`,
and a `use_model`. An optional `$comment` carries the SPDX header. The schema is
authoritative; this prose explains intent.

[#changes-from-0-1]
== Changes from 0.1

Version 0.2 (R-2026-07-07, ADR 0004) changes *verdict semantics only*; the wire
format is unchanged, byte-for-byte, from 0.1:

* *A1 — two-sided deceptive zeros.* `Dropped ▷ Falsified = Falsified` (the lie
  survives the drop); the deceptive duals are two-sided absorbing zeros of `▷`.
* *A2 — chain retention order.* The fate order is a CHAIN:
  `Falsified ⊏ Dropped ⊏ Predicated ⊏ Attenuated(δ) ⊏ Present` (the former
  `Predicated ∥ Dropped` antichain is resolved; `Dropped ⊑ Predicated` now holds).
* *A3 — `Attenuated(0)` ingest normalization.* The identification
  `Attenuated(0) = Present` is imposed by normalization at IR ingest: validators
  MUST rewrite `Attenuated(0)` to `Present` before grading, in edge grades and
  floors alike. `Attenuated(0)` stays wire-legal. Composition preserves normal
  forms (finite tropical addition reaches 0 only from 0 + 0), so no algebra
  change is entailed.

Because A1/A2 changed which documents pass a given floor with no wire-format
change, consumers see ONE semantic version bump covering all of R-2026-07-07.
Implementations MUST accept both `"0.1"` and `"0.2"` documents and MUST
normalize and grade them identically (under 0.2 semantics); any other `version`
MUST be rejected at validation with the coordinate named.

== Nodes — property-instances

A *node* is a property-instance carrying its `type` tag and (for tropes) its
field-presence over Φ = {quality, bearer, context, record}:

* `Trope` — a property-instance over a field-presence subset (`present`).
* `FloatingQuality` — a quality severed from its bearer; it MUST NOT list
  `bearer` in `present` (the severance is structurally visible, calculus §3.2).
* `Codomain` — a collapse target (e.g. `Bool`); a non-trope, carrying `codomain`.

== Edges — graded effect applications

An *edge* carries an `effect`, `inputs` (node ids), an `output` node, and a
`grade`. The six writable effects are `preserve`, `project`, `collapse`,
`detach`, `attenuate` (single-instance) and `fuse` (two-instance). The deceptive
trio (falsify, misbind, conflate) and any untagged merge are *not* writable
(HC-3) and cannot appear as a well-formed effect.

== The grade — a dependent record (HC-4)

A grade is `Σ(fate : Φ → Fate). Bond(fate.bearer) × Merge`. The schema encodes the
dependency so incoherent grades are *unrepresentable*:

[cols="1,3",options="header"]
|===
| Coordinate | Values (prevent profile)
| `fate` (per field) | `Present` · `Attenuated(δ)` with `δ ∈ ℕ∪{inf,top}` · `Predicated(P)` (quality only) · `Dropped`. (`Falsified` is the deceptive dual — excluded.)
| `bond` | `Intact` where the bearer is present; `Withheld`/`Severed` where absent. (`Misbound` is the deceptive dual — excluded.)
| `merge` | `Single`; or `Fused` with a REQUIRED provenance tag `tau`. (`Conflated` is the deceptive dual — excluded.)
|===

The `if/then/else` on `bond` keyed by `fate.bearer` is the door-too-small applied
to the schema itself: a grade with the bearer present yet `Severed`, or absent yet
`Intact`, does not validate, and the validator names the coordinate
(`edges/N/grade/bond`).

== The grade algebra (language-neutral, normative)

Stated here so two independent checker implementations agree by construction.

*Order convention.* `⊑` is the RETENTION order: `x ⊑ y` iff `y` retains at least
as much particularity as `x`. It is the componentwise product order; each
coordinate's order is in the calculus (§3, §5). The deceptive dual of each
coordinate is the bottom (worst retention), so every honest floor fails against it.

*Composition* `▷` (apply the second to the residual of the first) is componentwise
(calculus §4): tropical addition on the fidelity δ (`top` absorbs `inf` absorbs
finite); `Dropped`/`Severed`/`Conflated` (and the deceptive duals) absorbing;
`Present`/`Intact`/`Single` the units.

*Accumulation over the DAG.* Each root node has grade ε. For a single-input edge
`e` with source `s`: `acc(e.output) = acc(s) ▷ grade(e)`. For a two-input `fuse`
edge: `acc(e.output) = (acc(s1) ⊓ acc(s2)) ▷ grade(e)`, where `⊓` is the
retention-meet (the result is "no better than its worse constituent in any
coordinate", calculus T-Fuse).

== The verdict

Given the `use_model` (a designated `output` node and a partial `floor` grade):

[stem]
++++
\mathsf{p\text{-}sufficient} \iff \mathrm{floor} \sqsubseteq \mathrm{acc}(\text{output}).
++++

If it fails, the verdict is `p-insufficient` and the report carries a *witness*:
the topologically-first edge whose accumulated output grade drops below the floor
in a failing coordinate — the edge to repair (the trope-particularity analogue of
the invariant-path argmin).

== Profiles

* *prevent* (this schema, shipped): deceptive grades are malformed; rejected at
  validation with the coordinate named.
* *detect* (documented, calculus §6.1): deceptive grades are representable and
  caught at the verdict instead. An implementation MUST declare which profile it
  serves; this repository serves *prevent*.

== Conformance

`tests/conformance/cases.json` + `fixtures/` (IR → expected verdict, with witness)
and `schema-invalid/` (documents that MUST be rejected, with the coordinate named)
are the conformance contract. Any implementation MUST reproduce them. See
`docs/CONTRIBUTING-A-FRONTEND.adoc`.
