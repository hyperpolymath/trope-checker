= ADR 0002 — Trope-particularity checker build
:revdate: 2026-06-22

== Status

Accepted (alpha).

== Context

The trope-particularity calculus (`spec/calculus.adoc`) defines a graded effect
discipline over property-instances. This repository realises its *portable trust
boundary*: a checker from a language-neutral Trope IR to a verdict, plus the IR
schema, a verified core, and a conformance suite.

== Decisions

[cols="1,3",options="header"]
|===
| ID | Decision
| *R-2026-06-22 (deploy variant = prevent)* | Deception is a validation-boundary fault. The three deceptive grades (Falsified/Misbound/Conflated) and any untagged merge are not writable (HC-3) and are malformed in a valid Trope IR; the schema rejects them at validation, naming the coordinate. The `detect` variant (catch at the verdict) is documented but not shipped.
| *Schema is the trust boundary* | `schemas/trope-ir.schema.json` (draft 2020-12) is canonical. HC-4 (the Σ-dependency bond⊣bearer-fate) is encoded with `if/then/else` so incoherent grades are unrepresentable; deceptive grades are excluded from the prevent profile.
| *Two cores + conformance* | The Idris2 development (`verification/proofs/idris2/`) is the verified reference (it proves the laws); the Python `tropecheck.py` is the executable/shipping sketch; the conformance suite ties them. A Rust fast-core is deferred. The checker depends on no source language (HC-1).
| *RSR placement* | Built into the existing RSR-template repo, honouring its layout: calculus+IR spec → `spec/`; schema → `schemas/`; checker → `src/checker/`; conformance → `tests/conformance/`; proofs → `verification/proofs/idris2/`; status → `docs/status/` + `AFFIRMATION.adoc`. `spec/` and `schemas/` added to `.machine_readable/root-allow.txt` with justification.
| *Licensing* | Code/config/schemas = MPL-2.0; prose docs = CC-BY-SA-4.0 (two SPDX lines); single MPL-2.0 GitHub badge from root `LICENSE`. `LICENSES/` carries both texts.
| *Stub-and-name (HC-5)* | Grade soundness (`g ⊑ ρ`) is not asserted; it is the named open problem in `docs/status/PROOF-STATUS.adoc`. Only machine-checked lemmas are claimed proved.
|===

== Consequences

The checker is sound (not complete), relative to faithful lowering, and does not
check veridicality — these firewalls are stated in the README and never laundered.
A new front-end becomes "portable" by lowering to the IR and passing the
conformance suite (`docs/CONTRIBUTING-A-FRONTEND.adoc`).
